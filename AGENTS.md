# BuyerReach repository instructions

本仓库所有开发者和 AI 编程工具必须先阅读并遵循：

- `docs/DEVELOPMENT_RULES.md`（唯一权威开发规则）
- `docs/pipeline-production-architecture-v1.md`（Pipeline、状态机和运行架构）
- `docs/production-integrity-audit-2026-07-17.md`（当前能力边界和发布约束）

若编辑器专属规则与 `docs/DEVELOPMENT_RULES.md` 冲突，以后者为准。不得在编辑器专属文件中建立第二套架构规则。

每次完成修改后必须按变更风险执行验证，并在交付说明中报告实际执行的命令和结果。
