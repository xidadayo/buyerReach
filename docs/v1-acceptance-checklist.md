# V1 Acceptance Checklist

- AT-01: Create exact brand search task and produce progress records
- AT-02: One brand can store and display at least five contacts
- AT-03: Website public email extraction records source URL and captured time
- AT-04: Email verification records provider, raw result mapping, status, score, and time
- AT-05: Normalized duplicate emails do not create duplicate core records
- AT-06: Catch-all or uncertain email enters `risky` or manual review, not `valid`
- AT-07: Suppressed email/domain cannot enter valid pool
- AT-08: Failed task item can be retried with visible error reason
- AT-09: Sales role cannot export all full emails
- AT-10: Admin can export brand + contact + email + source + verification status
- AT-11: Backup can be restored in a clean environment
- AT-12: Provider can be replaced without changing brand/contact/email core services
