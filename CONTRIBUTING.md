# Contributing to BuyerReach

All contributors must read [the development rules](docs/DEVELOPMENT_RULES.md) before making changes.

The rules define architecture boundaries, Vendor governance, migrations, compatibility, security, testing, rollout, rollback, and the definition of done. Pull requests and editor-generated changes are reviewed against that single contract.

Do not copy the rules into editor settings. Update the canonical document through a reviewed change when the system architecture genuinely evolves.
